Personal Financial Tracker is a local-first personal finance app. It tracks your money on a single-entry, transaction-date basis — import bank activity, review and categorize it, then analyze it with pivots, budgets, and drill-downs.

The core loop

Import → Review → Approve → Analyze. Bank CSVs (checking + credit cards) upload via Tools → Import; rows land in the Review Queue with auto-categorization, vendor cleanup, duplicate flagging (overlapping date ranges are safe — dupes are caught against everything already imported), and auto-skip suggestions for CC payments/Venmo. Approve each row and it becomes an active transaction. Manual entry covers Venmo/cash. Every list supports inline editing (dates, category, amount, vendor, notes, tags), and mistakes are recoverable: deleted rows go to Trash, whole imports can be undone, splits can be un-split.

Structure & accuracy rules

Transactions carry a type (expense / income / transfer) and an L1 / L2 category (tree lives on the Categorization page — you can add L1s/L2s yourself there). Credits/refunds are negative expenses (they render green). Splits divide one bank charge into children that must sum exactly to the parent — useful for splitting a mixed charge across categories (button on any transaction page). Guard rails prevent the dangerous stuff: split children can't be deleted directly, and linked rows are protected.

Receipts

Drop files (photo, PDF, screenshot — incl. foreign-currency receipts) into the inbox folder, hit Scan inbox, then review matches one at a time with keyboard shortcuts. Nothing files without your confirm; confirming moves + renames the file into receipts/filed/<year>/<L1>/<L2>/<month>/ and saves the path. Duplicates get a red banner (discard to a recoverable trash, or keep as a second copy). Orphans (unmatched) get the same one-at-a-time review with vendor/amount (±10%) search. Foreign receipts match by date + the exchange rates set under Assumptions.

Analysis & tools

Dashboard (YTD cards, pending badges), Expenses/Income overview pivots (month × category, click into L2s), by-vendor drill-downs, Budget Values + Actuals vs. Budget, and an experimental Investments module (accounts, lots, transfers, returns). The Tools section holds the operational utilities — import, receipts, data cleanup (vendor/tag/duplicate/receipt hygiene, one item at a time) — each explained on the All Tools page; pick which show in the sidebar under Tools Menu.

Under the hood

Flask + SQLite, local-only, password login (change it under Security; recovery = the .app_password file next to the app). Your data lives entirely on your machine — back up the app folder (code, database, receipts) however you prefer, e.g. a private git repo or synced drive.
